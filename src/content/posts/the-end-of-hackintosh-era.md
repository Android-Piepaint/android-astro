# 前言
爲什麼要使用「黑蘋果」？或者講，爲什麼要使用 macOS 呢？</br>
這個世界總是那麼奇怪。人們對微軟的 Windows 作業系統所施加的種種流氓行爲怨聲載道，甚至拒用 Windows 系統。對於現今的 Google 所推出的 Android 限制應用程式安裝的規定也紛紛「聯合抵制」，還有人爲此致電政府，呼籲國家部門關注此事並且採取措施。</br>
但是，一旦作業系統或者裝置的製造廠家是 Apple，許多人反而變成了「沈默的思想者」，似乎都不再抵制了。史托曼曾經說過，iPhone 和 MacBook 是「監獄」，使用者卻心甘情願的接受來自 Apple 的牢籠，還稱之爲「時尚」，以自居「果粉」爲榮，宣揚[商品拜物教](https://zh.wikipedia.org/zh-tw/%E5%95%86%E5%93%81%E6%8B%9C%E7%89%A9%E6%95%99)。</br>
</br>
一般人被史蒂夫·賈伯斯（或者是現在的蒂姆·庫克）的話術說服，覺得iPhone，MacBook 很時尚很潮流，頗有極客風範。于是大家都不由自主地跑到 Apple store 去講：「請給我帶上手銬吧！」「工程師認爲 UNIX-based 的系統非常方便，所以犧牲自由和隱私是沒關係的，也給我一份手銬吧！」「你看我的手銬上還有我的名字呢！」每次我看到不少人都在推薦 Mac 時，我都會不由自主地想這些人是否是受虐狂，喜歡被束縛住手腳，然後把自己本該擁有的權利視爲「主人的恩賜」？這簡直和我喜歡看的H漫畫不謀而合！</br>
</br>
膜拜 Apple 的硬體就算了，甚至連作業系統這個「工具」都要去膜拜...爲了在普通電腦甚至一些嵌入式裝置安裝 macOS 而建立的 [Hackintosh 社區](https://zh.wikipedia.org/zh-tw/Hackintosh)就是一個典型的例子。</br>
當然，也不能講的如此刻板。畢竟黑蘋果社群裏也有一些開發者，他們研究 macOS 的組成，開發出不少 kext，讓大量非 Apple 官方認可甚至不被支援的硬體跑起來，亦有人開發 Clover 和 OpenCore 開機載入器用於研究 macOS 的機理。開發者們貢獻良多，沒準哪一天還真的可能會協助開源社群逆向出完全自由的 macOS 出來呢，例如 [ravynOS](https://github.com/ravynsoft/ravynos)就是採用 FreeBSD 爲基礎，混合 Darwin 核心所開發的相容 macOS 的作業系統，有點像 ReactOS 與 Windows 的關係。某種程度而言，黑蘋果社群就是推廣專有軟體和
自由軟體的結合體。不過堅定應援專有軟體的內容佔了大多數。</br>
</br>
2020年，Apple 開始往 ARM 架構遷移，就像他們在十幾年前，從 PowerPC 遷移至 Intel X86 那樣，逐步放棄支援 X86 架構的電腦和裝置。且根據 [Apple 和其它媒體的報道](https://www.pcmag.com/news/apple-confirms-end-of-support-for-intel-macs-after-macos-tahoe) 來看，macOS 26 應該是最後支援 X86 晶片的版本，之後的普通 X86 電腦就沒法玩黑蘋果了，黑蘋果已經死透，至少妳沒有辦法安裝最新版的 macOS 了。雖然可以安裝舊版，但是主流軟體也會慢慢捨棄支援的。現階段開源社群逆向 Apple Silicon 平臺的成果，除了能夠跑 Asahi Linux，或者使用修改的 `m1n1` 來載入 UEFI 韌體[引導 Windows 作業系統](https://github.com/AppleWOA)（或者其它支援 ACPI 的作業系統）。目前還沒有可以把 macOS 執行在 ARM 裝置上。</br>

當然，Windows 比起 macOS 也好不到哪去。至少，這樣可以讓人徹底斷掉 Hackintosh 的念頭，20年以來，除去探究機理和開發 kext 的，黑蘋果本身就是給大公司當「舔穴者(bootlicker)」的行爲。</br>

然後我再來講使用者的心態，靠着破解大公司的商業軟體來滿足自己的虛榮心。macOS 是閉源軟體，更是間諜軟體。它雖然很優秀，但請不要忘記
macOS 是竊取了 BSD 30年來革命果實的邪惡力量。這也怪 BSD，誰叫妳們不用更具有法律效益的 GPL，反而要使用[商業軟體友好，不講究開放原始碼和回饋上游的 BSD 協議](https://lukesmith.xyz/articles/why-i-use-the-gpl-and-not-cuck-licenses/)呢...</br>
</br>
套用《逆統戰》遊戲中的一句話：「娘係匹！BSD太過商業友好，看來我GPL協議必須出山！」</br>
</br>
macOS，使用者的自由處處受到 Apple 控制，除了軟體只能從 App store 下載之外，想要體驗到純正 UNIX 風的自由軟體，就只能藉助 Homebrew 這種殘廢套件管理員，或者是使用 macPorts 編譯老舊的軟體。還要忍受着 Apple 的「計劃性報廢」，時間到了就無法升級，哪怕硬體效能依舊。而 Apple 會說：「對不起，我們已經儘力支援很長一段時日了」...</br>
</br>
此時有的使用者會跳出來講：「macOS的設計美學非常不錯」，如果只是喜歡 macOS 的設計語言，爲什麼不使用 Linux 或者 FreeBSD，安裝 KDE / GNOME 桌面。使用 macOS 主題就可以，還可以還原80%的使用體驗？非要去安裝 Hackintosh，心甘情願的把自己關進「監獄」，實在匪夷所思，難以理解。有這些時間不如去研究如何在ESP32 單板機上移植 FreeBSD 或者去改善 SM8750 原型機的 Linux 核心支援。</br>
</br>
退一步講，即使要因爲工作或是學業要使用專有軟體，改用 Windows 不是比黑蘋果更好嗎？儘管 Windows 也是間諜軟體，至少還會給使用者基本的選擇自由，而非從頭到腳都被牢牢掌控。</br>
</br>

# 那麼，應該做什麼？

「**我們應該做的，是多研究怎麼樣在不自由的硬體上安裝 Linux，盡自身最大努力去逆向硬體驅動，研究機理，移植符合業界標準的開機載入器，從而達到解放本就屬於妳的裝置的目的，而不是散播甚至應援專有軟體和間諜軟體。**」</br>
就算 macOS 是 UNIX 也不要推薦。現在的 macOS 早就失去1980年代的駭客精神了。因此，移植 macOS 和 Windows on ARM 遠不如 Linux on everything 有價值（把 Linux 改造成 Android 那樣的「圍牆花園」則不算）。至於 MacBook 和 iPhone 是監獄，使用者心甘情願的給自己帶上手銬，還蔚然成風，以「果粉」而自豪。

# 當衆人都在爲個人電腦先驅者、最優秀的企業締造者的殞落而悲傷時，只有一個與衆不同的聲音大聲的喊道：「死的好！」
這個人就是理查·史托曼。2011年，Apple 的 CEO 史蒂夫·賈伯斯逝世，各界紛紛發文哀悼。可是自由軟體基金會主席史托曼卻說：「[死的好](https://stallman.org/archives/2011-jul-oct.html#06_October_2011_%28Steve_Jobs%29)。」

> 史蒂夫·賈伯斯，那位專門為笨蛋用戶製造酷炫監獄，剝奪人們自由的個人電腦先驅者，死了。...我並不為他的死而高興，但我很高興他走了。...沒有人是活該去死的...即使他犯下的惡果幾輩子也還不完。但是我們終究都會被賈伯斯對個人電腦的惡劣設計所影響。</br>
> ——節選自他的個人部落格

Apple產品借用了一部分BSD開源專案的成果，卻把系統變成不自由的軟體。這可以說是「自由軟體之敵」，Apple公司並不比Microsoft善良，所以理查·史托曼會這麼討厭Apple也就不奇怪了。他是對自己的信念堅持到底的一個人物。[^1]</br>

Apple公司對用戶的控制，有增無減，所謂的Apple生態系，實則是將用戶困入更深的牢籠中無法脫身。

如果你本身就有相對自由的硬體（X86不錯，ARM更好，RISC-V更棒），而且還有自由的Linux與BSD作業系統可選，爲什麼一定要去應援專有軟體，把自己關進專有軟體鑄成的監獄？
